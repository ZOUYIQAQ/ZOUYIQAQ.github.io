正规方程

原理

这个十分好理解，在计算线性回归问题时，会产生一个随机的线性方程，比如这样的：

这是随机产生的，那这个方程毫无疑问会和真实情况有所差距，比如真实的方程长这样：

而现在我们想要将这个随机出来的方程尽可能的靠近真实的情况，只要靠的越近，那预测的效果就越好。

正规方程就是用来缩小差距并一步到位的方法，它会通过一系列的复杂计算，然后直接得出最后结果（方程中的k和b），得到真实最接近真实方程的方程。

这篇专栏讲的十分详细，有需要的可以看看。

使用

LinearRegression的可选参数

  • fit_intercept：布尔型，默认值为 True。表示是否计算此模型的截距。如果设置为 False，则计算中不使用截距（即数据应居中）。
  • normalize：布尔型，默认值为 False。当 fit_intercept 设置为 False 时，将忽略此参数。
  • copy_X：布尔型，默认值为 True。如果为 True，则 X 将被复制；否则，它可能会被覆盖。
  • n_jobs：整型，默认值为 None。用于计算的作业数。这只有在问题足够大的情况下才会提供加速，即首先 n_targets > 1，其次 X 是稀疏的或者 positive 被设置为 True。None 表示 1，除非在 joblib.parallel_backend 上下文中。-1 表示使用所有处理器。
  • positive：布尔型，默认值为 False。当设置为 True 时，强制系数为正数。此选项仅支持密集数组。

补充

优点

  1. 建模速度快，不需要很复杂的计算。
  2. 可以根据系数给出每个变量的理解和解释。
  3. 线性回归模型十分容易理解，结果具有很好的可解释性，有利于决策分析。
  4. 蕴含机器学习中的很多重要思想。
  5. 能解决回归问题。

缺点

  1. LinearRegression 对数据存在过拟合的风险。为了避免过拟合，可以使用更强约束的模型，如 Ridge 回归。
  2. 线性回归要求因变量是连续性数值变量，而且要求自变量和因变量呈线性关系。
  3. 因为要计算矩阵的转置，有极高的时间复杂度，在大数据条件下速度过慢。

梯度下降

原理

其实和正规方程是差不多的，只是正规方程是一步到位直接算出参数，而梯度下降则是一步步的逼近真实值。其在大数据条件下也能有不错的表现。

可以参考这篇文章：梯度下降算法（附代码实现） - 知乎 (zhihu.com)

使用

SGDRegressor的可选参数

  • loss: 要使用的损失函数。可能的值是 ‘squared_loss’, ‘huber’, ‘epsilon_insensitive’ 或 ‘squared_epsilon_insensitive’。默认值为 ‘squared_loss’。
  • penalty: 要使用的惩罚（也称为正则化项）。可能的值是 ‘l2’, ‘l1’, ‘elasticnet’ 或 None。默认值为 ‘l2’。
  • alpha: 乘以正则化项的常数。值越高，正则化越强。默认值为 0.0001。
  • l1_ratio: Elastic Net 混合参数，0 <= l1_ratio <= 1。l1_ratio=0 对应 L2 惩罚，l1_ratio=1 对应 L1。仅在 penalty 为 ‘elasticnet’ 时使用。默认值为 0.15。
  • fit_intercept: 是否应该估计截距。如果为 False，则假设数据已经居中。默认值为 True。
  • max_iter: 训练数据（也称为纪元）的最大传递次数。它只影响 fit 方法中的行为，而不影响 partial_fit 方法。默认值为 1000。
  • tol: 停止标准。如果不是 None，训练将在 (loss > best_loss - tol) for n_iter_no_change 连续 epoch 时停止。默认值为 0.001。
  • shuffle: 训练数据是否应该在每个 epoch 之后重新洗牌。默认值为 True。
  • epsilon: Epsilon 在对 epsilon 不敏感的损失函数中；仅当 loss 为 ‘huber’, ‘epsilon_insensitive’ 或 ‘squared_epsilon_insensitive’ 时使用。默认值为 0.1。
  • learning_rate: 学习率表：‘constant’, ‘optimal’, ‘invscaling’ 或 ‘adaptive’。默认值为 ‘invscaling’。
  • eta0: “常数”，“Invscaling”或“自适应”计划的初始学习率。默认值是0.01。
  • power_t: 逆缩放学习率的指数。默认值为 0.25。
  • early_stopping: 当验证分数没有提高时，是否使用提前停止来终止训练。如果设置为 True，它会自动留出一小部分训练数据作为验证，并在 score 方法返回的验证分数在 n_iter_no_change 连续epoch中没有提高至少 tol 时终止训练。默认值为 False。
  • validation_fraction: 预留作为早期停止验证集的训练数据的比例。必须介于 0 和 1 之间。仅当 early_stopping 为 True 时才使用。默认值为 0.1。
  • n_iter_no_change: 在提前停止之前等待没有改进的迭代次数。默认值为 5。

补充

优点

  • 计算效率高。由于每次只选取一个样本进行计算，因此计算速度快。
  • 可以处理大规模数据集。由于不需要将所有样本都进行计算，因此可以处理大规模数据集。

缺点

  • 准确度下降。由于单个样本并不能代表全体样本的趋势，因此可能会收敛到局部最优。
  • 可能会受到噪声数据的影响。由于每次只选取一个样本进行计算，因此如果该样本是噪声数据，则会对结果产生较大影响。

岭回归

原理

在线性回归的基础上增加了正则化以降低部分参数对模型的影响从而达到降低过拟合的效果。

具体可以参考这篇博客，有较为详细的说明：岭回归原理简单分析与理解_岭回归分析_0不觉的博客-CSDN博客

使用

Ridge的可选参数

  • alpha：正则化系数，较大的值指定更强的正则化。
  • fit_intercept：是否计算该模型的截距。
  • normalize：是否对数据进行标准化处理。
  • copy_X：是否对X复制。
  • max_iter：共轭梯度求解器的最大迭代次数。
  • tol：计算精度。
  • solver：求解器，可选值有 {auto,svd,cholesky,lsqr,sparse_cg,sag,saga}。
  • random_state：随机数生成器的种子。

补充

优点

  • 通过调整L2惩罚因子，防止维度变高时参数过大问题1。
  • 它可以处理多重共线性问题，即当自变量之间存在高度相关性时，它仍然可以给出稳定的结果2。

缺点

  • 参数不能为0，即无用参数不能被削减1。
  • 它会保留所有变量，这会使得模型的变量特别多，模型解释性差3。
